# Politique de sécurité - Ba7ath Project

## Signaler une vulnérabilité de sécurité

Si tu as découvert une faille de sécurité dans ba7ath_project :

### Processus de signalement

1. **NE PAS** créer d'issue publique sur GitHub
2. Utilise **GitHub's Private Vulnerability Reporting**
3. Ou contacte-nous directement via le formulaire privé de signalement

### Détails à inclure dans ton rapport

- Description de la vulnérabilité
- Étapes pour reproduire le problème
- Impact potentiel
- Versions affectées
- Solutions proposées (si applicable)

## Processus de correction

- **Réponse** : dans les 3 jours ouvrables
- **Failles critiques** : patch de sécurité dans les 7 jours
- **Failles moyennes** : patch de sécurité dans les 14 jours
- **Failles faibles** : intégrées dans la prochaine version

## Versions supportées

| Version | Supportée | État |
|---------|-----------|------|
| 1.0.x   | ✅ Oui   | Actif |
| < 1.0   | ❌ Non   | EOL |

## Protocole de divulgation responsable

- Les failles seront corrigées **avant** la divulgation publique
- Les chercheurs en sécurité peuvent être crédités (sur demande)
- Un délai de 90 jours est respecté avant divulgation publique
- Coordination avec d'autres projets affectés si nécessaire

## Reconnaissance

Nous apprécions les rapports de sécurité responsables. Les chercheurs en sécurité qui signalent des vulnérabilités seront crédités dans :
- Les notes de version
- Notre page de remerciements
- Les annonces de sécurité

## Questions ou doutes ?

Pour toute question sur cette politique :
- Utilise GitHub's Private Vulnerability Reporting
- Consulte notre documentation de sécurité
- Ouvre une discussion privée sur le repo
